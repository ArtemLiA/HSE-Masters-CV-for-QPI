# Описание

В репозитории хранятся исходный код и данные для экспериментов, проводимых в рамках выполнения курсовой работы на 1-ом курсе магистратуры "Финансовые технологии и анализ данных" НИУ ВШЭ


# Структура проекта

- `src` - исходный код
- `data` - данные для экспериментов
    - `data/stamps` - датасет из 50 документов с печатями
    - `data/red-blood-cells`


## Исходный код

- `src`
    - `src/detector.py` - классы детектора и предобработчика изображения
    - `src/fourier.py` - реализация алгоритма на основе Фурье-преобразования для восстановления фазы
    - `src/plotting.py` - инструменты для визуализации
    - `src/geometry` - пакет для работы с геометрией: хранение объектов, вычисление метрик, мэтчинг, загрузка данных


## Формат данных

Данные для каждой задачи хранятся в отдельной директории, которая состоит из:
- `annotation.json` - JSON-файл с размеченными границами объектов
- `images/` - директория с изображениями в формате png


JSON-файл имеет следующую структуру:
```json
{
    "image_id": {
        "label class": [
            {
                "x": ..., 
                "y": ..., 
                "rx": ..., 
                "ry": ..., 
                "angle": ...,
                "width": ...,
                "height" 
            }
        ]
    } 
}
```

где 
- `image_id` - идентификатор изображения (например, `"img001"`)
- `label class` - класс, к которому относятся объекты из списка-значения
- `x`- x-координата центра объекта
- `y`- y-координата центра объекта
- `rx` - абсолютный радиус по x в пикселях
- `ry` - абсолютный радиус по y в пикселях
- `angle` - угол наклона (во всех случаях 0)
- `width` - ширина изображения в пикселях
- `height` - высота изображения в пикселях

Для удобства работы с Label Studio все объекты изначально размечались в виде эллипсов. 

Для всех объектов `rx`, `ry` практически совпадают
